# Data Visualisation Catalogue Selection Guide

Use this reference when choosing chart types for stakeholder briefs. The default external guide is The Data Visualisation Catalogue:

- Main catalogue: `https://datavizcatalogue.com/index.html`
- Search by function: `https://datavizcatalogue.com/search.html`

The catalogue organizes visualization choice by what the chart must communicate. Use that function-first approach before choosing a chart type.

## Selection Process

1. State the slide message in one sentence.
2. Identify the communication function.
3. Shortlist chart types from the catalogue function.
4. Pick the simplest chart that supports the decision.
5. Record the function, chart type, and why it was chosen in `presentation-generator-brief.md`.

## Function Map

Use the catalogue's function categories as the default vocabulary:

| Communication Need | Catalogue Function | Common Executive Choices |
| --- | --- | --- |
| Compare values across categories | Comparisons | Bar chart, multi-set bar chart, bullet graph, heatmap |
| Show shares or relative size | Proportions, Part-to-a-whole | Stacked bar, dot matrix, treemap, proportional area, donut only for very few parts |
| Show correlation or association | Relationships | Scatterplot, heatmap, bubble chart |
| Show hierarchy or nested structure | Hierarchy | Tree diagram, treemap, sunburst |
| Explain an idea or model | Concepts | Flow chart, illustration diagram, Venn diagram |
| Show geography | Location | Choropleth map, dot map, bubble map, connection map |
| Show distribution or spread | Distribution | Histogram, box and whisker plot, density plot, violin plot |
| Explain how something works | How things work | Flow chart, illustration diagram, network diagram |
| Show process steps or methods | Processes and methods | Flow chart, timeline, Gantt chart |
| Show movement or transfer | Movement or flow | Sankey diagram, flow map, chord diagram |
| Show repeated patterns | Patterns | Heatmap, calendar, spiral plot |
| Show ranges or uncertainty | Range | Span chart, error bars, box and whisker plot |
| Show change over time | Data over time | Line graph, area graph, calendar, timeline, stacked area only when composition over time matters |
| Analyze text | Analysing text | Word cloud only for exploratory emphasis; prefer ranked bars for decisions |
| Provide a lookup or schedule | Reference tool | Table, timetable, calendar |

## Executive Deck Biases

- Prefer bar, line, dot/lollipop, heatmap, scatterplot, annotated screenshot, and small multiples for most business decks.
- Use Sankey, chord, treemap, sunburst, radar, network, and radial charts only when their structure is essential to the message.
- Avoid pie and donut charts unless there are very few categories and exact comparison is not important.
- Avoid word clouds for decision-making unless paired with a ranked table or bar chart.
- For top/bottom rankings, use sorted bars or dot/lollipop charts with denominator thresholds.
- For funnels and ordered drop-off, use a funnel, stepped bar, or line/area over stages; show denominator and stage definition.
- For UX, page, dashboard, map, or workflow analysis, combine the chart with recognizable source context when allowed.
